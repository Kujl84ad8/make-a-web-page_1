Lesson 2: Creating a Structured Document with HTML
Developer Tools (in the Browser)
-Block elements form an "invisible box" around the content inside of them

-The "tree-like structure" of HTML - The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes.

-The relationship between indented HTML and boxes When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Lesson 3: Adding CSS Style to HTML Structure
Avoid Repetition
-Avoiding repetition is important for a variety of reasons.

-Errors: If you are hard coding each line of text eventually errors will be made.

-Ease of Change: If you change your mind on how you want the page to look you can make that change in one place and not hundreds.

-Consistency: Using CSS makes your entire page flow and look consistent

CSS
-These provides a template to control many HTLM elements. By giving elements the same "class" name all of the CSS attributes will be applied to that element.
